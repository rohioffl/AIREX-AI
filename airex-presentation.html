Security Architecture

Safety-critical from day one.

🔐 RBAC + Permissions
  • 3 roles: VIEWER < OPERATOR < ADMIN
  • Granular permissions: incident:approve, incident:senior_approve, user:* …
  • require_permission() on every route — no role string checks
  • Senior approval required for HIGH-risk actions
🛡️ Database & RLS
  • PostgreSQL Row-Level Security on all tables
  • Composite PKs: (tenant_id, id) everywhere
  • nullable=True on tenant_id is banned
  • Parameterized queries only — no raw SQL
🔗 Immutable Audit Chain
  • state_transitions table: never UPDATE or DELETE
  • SHA-256 hash chain across every transition record
  • Structlog on every log line with correlation_id + tenant_id
  • Langfuse trace on every LLM call
⚡ Request Hardening
  • HMAC-SHA256 webhook signature verification
  • Idempotency keys prevent duplicate incident creation
  • Rate limiting on all public endpoints
  • LLM-generated shell commands strictly prohibited
